LV2 is the PS3's hypervisor and kernel layer. Oxidized-cell provides high-level emulation (HLE) of LV2 system calls for process management, threading, synchronization, file I/O, and time management.

## Overview

The LV2 component (`oc-lv2`) implements:

- **Syscall handling**: 400+ PS3 system calls
- **Process management**: Process creation and lifecycle
- **Thread management**: Thread scheduling and synchronization
- **Kernel objects**: Mutexes, semaphores, condition variables, event flags
- **File system**: Virtual file system operations
- **Time management**: System time and timers

```rust
pub use objects::ObjectManager;
pub use process::ProcessManager;
pub use syscall::SyscallHandler;
pub use thread::ThreadManager;
```

## Architecture

LV2 is organized into modules:

- **syscall**: Syscall dispatch and handler registration
- **syscall_numbers**: Syscall ID constants
- **process**: Process management and lifecycle
- **thread**: Thread creation and scheduling
- **objects**: Kernel object management
- **sync**: Synchronization primitives
- **fs**: File system operations
- **memory**: Memory management syscalls
- **time**: Time and timer syscalls
- **timer**: Timer objects
- **spu**: SPU management syscalls
- **prx**: PRX (shared library) loading

## Syscall handler

The `SyscallHandler` dispatches syscalls from PPU code:

```rust
use oc_lv2::SyscallHandler;

let mut handler = SyscallHandler::new(
    memory.clone(),
    object_manager.clone(),
);

// Register syscall handlers
handler.register_all();

// Execute syscall
let result = handler.handle(
    syscall_id,
    &mut ppu_thread,
)?;
```

### Syscall execution

1. **PPU executes** `sc` instruction with syscall ID in r11
2. **Trap to emulator** with syscall number and arguments in r3-r10
3. **Lookup handler** for syscall ID
4. **Execute handler** with thread context
5. **Return result** in r3, error code in r4

<Note>
Syscalls follow the PS3 ABI: arguments in r3-r10, return value in r3.
</Note>

## Process management

The `ProcessManager` handles process lifecycle:

```rust
use oc_lv2::ProcessManager;

let mut pm = ProcessManager::new();

// Create process from ELF
let process_id = pm.create_process(
    elf_data,
    args,
    env,
)?;

// Load process into memory
pm.load_process(process_id, &mut memory)?;

// Get process info
let info = pm.get_process_info(process_id)?;
```

### Process structure

<Accordion title="Process components">
- **Process ID**: Unique identifier
- **Entry point**: Main function address
- **Memory map**: Code, data, heap, stack regions
- **Threads**: List of threads in process
- **PRX modules**: Loaded shared libraries
- **File descriptors**: Open file handles
- **Working directory**: Current directory path
</Accordion>

## Thread management

The `ThreadManager` handles thread creation and scheduling:

```rust
use oc_lv2::ThreadManager;

let mut tm = ThreadManager::new();

// Create thread
let thread_id = tm.create_thread(
    entry_point,
    stack_size,
    priority,
    name,
)?;

// Start thread execution
tm.start_thread(thread_id)?;

// Wait for thread to complete
tm.join_thread(thread_id)?;
```

### Thread states

```
[Created] --> [Ready] --> [Running] --> [Terminated]
               ^   |           |
               |   v           v
               [Blocked]   [Waiting]
```

### Thread priority

PS3 thread priority ranges from 0 (highest) to 3071 (lowest):

| Priority | Description |
|----------|-------------|
| 0-255 | Real-time priority |
| 256-767 | High priority |
| 768-2047 | Normal priority |
| 2048-3071 | Low priority |

## Synchronization primitives

### Mutex

Mutual exclusion lock:

```rust
// sys_mutex_create
let mutex_id = syscall_handler.handle(
    80, // sys_mutex_create
    &mut thread,
)?;

// sys_mutex_lock (blocking)
syscall_handler.handle(
    81, // sys_mutex_lock
    &mut thread,
)?;

// sys_mutex_unlock
syscall_handler.handle(
    82, // sys_mutex_unlock
    &mut thread,
)?;
```

### Semaphore

```rust
// sys_semaphore_create

    94, // sys_semaphore_create

    95, // sys_semaphore_wait
    &mut thread,

// sys_semaphore_post (increment)

    96, // sys_semaphore_post
```

### Condition variable

Thread notification:

```rust
// sys_cond_create
let cond_id = syscall_handler.handle(
    100, // sys_cond_create
```
