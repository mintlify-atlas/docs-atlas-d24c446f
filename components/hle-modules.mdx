Oxidized-cell provides high-level emulation (HLE) of PS3 system libraries, allowing games to call PS3 APIs without requiring the actual PS3 firmware libraries in memory.

## Overview

The HLE component (`oc-hle`) implements:

- **Module registry**: Registration and lookup of HLE modules
- **Function dispatch**: Route game calls to HLE implementations
- **Memory utilities**: Read/write guest memory safely
- **Context management**: Track HLE subsystem state

```rust
pub use module::ModuleRegistry;
pub use dispatcher::{HleDispatcher, HleCallContext, HleFunctionInfo};
pub use context::{HleContext, get_hle_context, set_rsx_bridge};
pub use memory::{read_be32, write_be32, read_string};
```

## Module registry

The `ModuleRegistry` manages HLE modules:

```rust
use oc_hle::ModuleRegistry;

let mut registry = ModuleRegistry::new();

// Register all built-in modules
registry.register_all();

// Look up function by NID
let func_info = registry.lookup_function(
    module_name,
    nid,
)?;

// Get module info
let module_info = registry.get_module(module_name)?;
```

### Module structure

<Accordion title="Module components">
- **Name**: Module identifier (e.g., "cellGcmSys")
- **Version**: Module version
- **Functions**: List of exported functions
- **Variables**: List of exported variables
- **Dependencies**: Required modules
</Accordion>

## Function dispatch

The `HleDispatcher` routes game calls to HLE implementations:

```rust
use oc_hle::{HleDispatcher, dispatch_hle_call};

// Initialize dispatcher
let dispatcher = HleDispatcher::new();

// Dispatch function call
let result = dispatch_hle_call(
    module_name,
    nid,
    &mut ppu_thread,
)?;
```

### Call context

The `HleCallContext` provides access to:

```rust
pub struct HleCallContext<'a> {
    pub thread: &'a mut PpuThread,
    pub memory: Arc<MemoryManager>,
    pub rsx_bridge: Option<RsxBridgeSender>,
    pub spu_bridge: Option<SpuBridgeSender>,
}
```

## Memory utilities

HLE functions use memory utilities to access guest memory:

<Tabs>
  <Tab title="Read">
    ```rust
    use oc_hle::{read_be32, read_be64, read_string};
    
    // Read 32-bit big-endian value
    let value = read_be32(addr)?;
    
    // Read 64-bit big-endian value
    let value64 = read_be64(addr)?;
    
    // Read null-terminated string
    let text = read_string(addr)?;
    ```
  </Tab>
  <Tab title="Write">
    ```rust
    use oc_hle::{write_be32, write_be64, write_string};
    
    // Write 32-bit big-endian value
    write_be32(addr, value)?;
    
    // Write 64-bit big-endian value
    write_be64(addr, value64)?;
    
    // Write null-terminated string
    write_string(addr, "Hello")?;
    ```
  </Tab>
  <Tab title="Check">
    ```rust
    use oc_hle::{is_valid_pointer, is_readable, is_writable};
    
    // Check if pointer is valid
    if is_valid_pointer(addr) {
        // Safe to access
    }
    
    // Check read permission
    if is_readable(addr, size) {
        // Can read
    }
    
    // Check write permission
    if is_writable(addr, size) {
        // Can write
    }
    ```
  </Tab>
</Tabs>

<Warning>
Always validate guest pointers before dereferencing to prevent crashes.
</Warning>

## NID system

PS3 uses Name IDs (NIDs) to identify functions:

```rust
// Function NID (SHA-1 hash of name)
const CELL_GCM_INIT: u32 = 0x15BAE46B;
const CELL_GCM_SET_FLIP_MODE: u32 = 0xB2E761D4;
```

NIDs are 32-bit hashes derived from function names, allowing symbol resolution without strings.

## Implementation example

Example HLE function implementation:

```rust
// cellGcmInit implementation
fn cell_gcm_init(
    ctx: &mut HleCallContext,
) -> Result<i32, HleError> {
    // Read arguments from registers
    let io_address = ctx.thread.gpr[3];
    let io_size = ctx.thread.gpr[4];
    let control_addr = ctx.thread.gpr[5];
    let label_addr = ctx.thread.gpr[6];
    
    // Initialize graphics system
    let gcm_ctx = initialize_gcm(
        io_address,
        io_size,
        control_addr,
        label_addr,
    )?;
    
    // Send command to RSX
    if let Some(ref mut bridge) = ctx.rsx_bridge {
        bridge.send(BridgeCommand::Init(gcm_ctx))?;
    }
    
    // Return success (0)
    Ok(0)
}
```

## Adding custom modules

You can register custom HLE modules:

```rust
use oc_hle::{ModuleRegistry, HleFunctionInfo};

let mut registry = ModuleRegistry::new();

// Register custom module
registry.register_module(
    "myModule",
    vec![
        HleFunctionInfo {
            name: "myFunction",
            nid: 0x12345678,
            handler: my_function_handler,
        },
    ],
)?;
```
