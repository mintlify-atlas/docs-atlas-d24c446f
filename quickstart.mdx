This guide will help you run your first PS3 game with oxidized-cell. Follow these steps to go from installation to playing.

<Note>
  Before starting, make sure you have [installed oxidized-cell](/installation) on your system.
</Note>

## Running the emulator

Launch oxidized-cell from the command line:

<CodeGroup>
```bash Linux/macOS
# Navigate to the oxidized-cell directory
cd oxidized-cell

# Run the emulator
./target/release/oxidized-cell
```

```powershell Windows
# Navigate to the oxidized-cell directory
cd oxidized-cell

# Run the emulator
.\target\release\oxidized-cell.exe
```
</CodeGroup>

This launches the main UI window with the game list view.

## Running your first game

<Steps>
  <Step title="Choose a game format">
    oxidized-cell supports several PS3 file formats:

    | Format | Extension | Description |
    |--------|-----------|-------------|
    | **ELF** | `.elf` | Unencrypted executables (homebrew) |
    | **SELF** | `.self` | Encrypted executables (requires firmware) |
    | **ISO** | `.iso` | Disc images |
    | **PKG** | `.pkg` | PlayStation packages |
    | **PRX** | `.prx` | Shared libraries |

    <Note>
      Start with homebrew ELF files - they don't require firmware installation and are easier to test with.
    </Note>
  </Step>

  <Step title="Launch via command line">
    The simplest way to run a game is from the command line:

    ```bash
    cargo run --release -- /path/to/game.elf
    ```

    Or if you've built the release binary:

    ```bash
    ./target/release/oxidized-cell /path/to/game.elf
    ```

    The emulator will:
    1. Load the ELF file
    2. Initialize the Cell processor components (PPU, SPU)
    3. Set up the RSX graphics subsystem
    4. Begin execution
  </Step>

  <Step title="Launch via UI">
    You can also load games through the user interface:

    1. Launch oxidized-cell without arguments
    2. Click **File → Open** in the menu bar
    3. Navigate to your game file
    4. Click **Open**

    Or set up a games directory:

    1. Go to **Settings → Paths**
    2. Set **Games Directory** to your PS3 games folder
    3. Games will automatically appear in the game list
    4. Double-click a game to launch it
  </Step>

  <Step title="Configure basic settings">
    Before running demanding games, configure the emulator for optimal performance:

    ```bash
    # Create a config.toml file in the oxidized-cell directory
    ```

    ```toml config.toml
    [cpu]
    ppu_decoder = "jit"        # Use JIT for better performance
    spu_decoder = "jit"        # Use JIT for SPU as well

    [graphics]
    backend = "vulkan"         # Vulkan rendering backend
    resolution_scale = 1       # Native resolution (increase for upscaling)

    [audio]
    backend = "cpal"           # Cross-platform audio library
    volume = 100               # Volume level (0-100)
    ```

    Or configure through the UI:
    - **Settings → CPU** - Set decoders to "Recompiler" (JIT)
    - **Settings → Graphics** - Adjust resolution and rendering options
    - **Settings → Audio** - Configure audio output
  </Step>
</Steps>

## Understanding decoder modes

The emulator offers two execution modes for the PowerPC (PPU) and SIMD (SPU) processors:

### Interpreter mode

```toml config.toml
[cpu]
ppu_decoder = "interpreter"
spu_decoder = "interpreter"
```

**Pros:**
- More accurate and compatible
- Better for debugging
- Simpler implementation with fewer bugs

**Cons:**
- Significantly slower (2-10x)
- Higher CPU usage

**Use when:** A game doesn't work with JIT or you're debugging issues

### JIT (Recompiler) mode

```toml config.toml
[cpu]
ppu_decoder = "jit"
spu_decoder = "jit"
```

**Pros:**
- Much faster execution (5-10x speedup)
- Uses LLVM for optimized native code generation
- Better performance on complex games

**Cons:**
- Slightly less accurate
- May have compatibility issues

**Use when:** You want the best performance (recommended for most games)

<Note>
  The JIT compiler translates PowerPC instructions to x86-64 (or ARM64 on Apple Silicon) using LLVM. It applies O2 optimization passes for maximum performance.
</Note>

## Testing the emulator

Here's a complete example of running a homebrew game:

<Steps>
  <Step title="Download a homebrew application">
    Find PS3 homebrew applications from sources like:
    - [PSX-Place Forums](https://www.psx-place.com/forums/ps3-homebrew/)
    - [Homebrew Database](https://store.brewology.com/)

    Look for `.elf` or `.self` files.
  </Step>

  <Step title="Run with default settings">
    ```bash
    ./target/release/oxidized-cell ~/Downloads/homebrew.elf
    ```
  </Step>

  <Step title="Monitor execution">
    Watch the console output for emulation status:

    ```
    [INFO] Loading ELF file: homebrew.elf
    [INFO] Initializing PPU with interpreter decoder
    [INFO] Initializing 6 SPU threads
    [INFO] RSX: Vulkan backend initialized
    [INFO] Audio: cpal backend ready (48kHz, 2 channels)
    [INFO] Starting emulation...
    ```
  </Step>

  <Step title="Check performance">
    Press **F12** to toggle the performance overlay showing:
    - FPS (frames per second)
    - Frame time (milliseconds)
    - PPU/SPU utilization
  </Step>
</Steps>

## Common issues

### Game won't start

<Tabs>
  <Tab title="Check file format">
    Verify the file is a supported format:

    ```bash
    file game.elf
    # Should show: ELF 64-bit MSB executable, 64-bit PowerPC
    ```
  </Tab>

  <Tab title="Enable verbose logging">
    Run with detailed logs:

    ```bash
    RUST_LOG=debug cargo run --release -- game.elf
    ```

    Check the output for error messages about missing dependencies or unsupported instructions.
  </Tab>

  <Tab title="Try interpreter mode">
    If JIT fails, fall back to interpreter:

    ```toml config.toml
    [cpu]
    ppu_decoder = "interpreter"
    spu_decoder = "interpreter"
    ```
  </Tab>
</Tabs>

### Poor performance

<Steps>
  <Step title="Enable JIT compilers">
    Make sure both decoders are set to JIT mode in `config.toml`
  </Step>

  <Step title="Check GPU drivers">
    Verify Vulkan is working:

    ```bash
    vulkaninfo | grep "Device Name"
    ```
  </Step>

  <Step title="Reduce resolution scale">
    Start with native resolution:

    ```toml config.toml
    [graphics]
    resolution_scale = 1
    ```
  </Step>
</Steps>

### Graphics not displaying

<Warning>
  Ensure your GPU supports Vulkan 1.2 or later and drivers are up to date.
</Warning>

Test Vulkan support:

```bash
# Linux
vulkaninfo

# Windows (PowerShell)
vulkaninfo.exe

# macOS
vulkaninfo
```

If Vulkan is not working, check:
- GPU drivers are installed and current
- Vulkan SDK is installed
- `VK_ICD_FILENAMES` environment variable points to your ICD
