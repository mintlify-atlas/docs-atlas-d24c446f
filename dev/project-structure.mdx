## Overview

oxidized-cell uses a hybrid Rust/C++ architecture organized as a Cargo workspace. The Rust code handles core emulation logic, system management, and UI, while C++ provides high-performance JIT compilation using LLVM.

<Info>
The project follows a modular design where each major component is isolated into its own crate with well-defined interfaces.
</Info>

## Repository structure

```
oxidized-cell/
├── crates/              # Rust workspace crates
├── cpp/                 # C++ performance components
├── src/                 # Main binary entry point
├── examples/            # Example programs
├── scripts/             # Build and utility scripts
├── docs/                # Project documentation
├── .cargo/              # Cargo configuration
├── .github/             # CI/CD workflows
├── Cargo.toml           # Workspace manifest
├── rust-toolchain.toml  # Rust version specification
└── build-windows.sh     # Windows cross-compilation script
```

## C++ performance layer

Location: `cpp/`

### Build system

```
cpp/
├── CMakeLists.txt       # CMake configuration
├── include/
│   └── oc_ffi.h        # FFI header for Rust
└── src/
    ├── ffi.cpp         # FFI implementations
    ├── ppu_jit.cpp     # PPU JIT (LLVM)
    ├── spu_jit.cpp     # SPU JIT (LLVM)
    ├── rsx_shaders.cpp # SPIR-V shader compilation
    ├── atomics.cpp     # 128-byte atomic operations
    ├── dma.cpp         # DMA optimizations
    └── simd_avx.cpp    # AVX2 helpers (x86_64 only)
```

### CMake configuration highlights

```cmake
set(CMAKE_CXX_STANDARD 20)

# LLVM components for JIT
llvm_map_components_to_libnames(LLVM_LIBS
    Core ExecutionEngine MCJIT OrcJIT
    Support Target PowerPC X86 AArch64
    TransformUtils Analysis Passes
)

# Platform optimizations
if(CMAKE_SYSTEM_PROCESSOR MATCHES "x86_64")
    add_compile_options(-mavx2 -mbmi2)
endif()
```

### JIT compilation

The JIT compilers use LLVM IR generation:

<Steps>
  <Step title="Decode instruction">
    Parse PowerPC/SPU instruction bytes
  </Step>
  <Step title="Generate LLVM IR">
    Translate to LLVM intermediate representation
  </Step>
  <Step title="Optimize">
    Apply O2-level optimization passes
  </Step>
  <Step title="Execute">
    JIT compile to native code and execute
  </Step>
</Steps>

**Supported instructions:**
- PPU: 20+ PowerPC instructions
- SPU: 15+ SIMD instructions

## Data flow

<Steps>
  <Step title="Game loading">
    User selects game → `oc-vfs` mounts ISO/PKG → `oc-loader` parses ELF/SELF → Load into `oc-memory`
  </Step>
  <Step title="Execution">
    `oc-ppu` fetches instructions → Interpret or JIT compile → Execute on PPU thread
  </Step>
  <Step title="Graphics">
    Game calls `cellGcmSys` (HLE) → `oc-rsx` processes commands → Vulkan rendering
  </Step>
  <Step title="Audio">
    Game writes audio data → `oc-audio` buffers samples → `cpal` outputs to hardware
  </Step>
  <Step title="Input">
    User input → `oc-input` maps to PS3 controller → Game reads via `cellPad` (HLE)
  </Step>
</Steps>
